# 複数PC（自宅・会社）でのGitHubデータ同期マニュアル
（3月1日 制作）

このマニュアルは、自宅のPCと会社のPC間で、GitHubを使って安全にデータを同期（引き継ぎ）するための手順書です。

## 📍 1. 全体像（GitHubの仕組み）
GitHubは**「インターネット上の金庫」**です。
USBメモリを持ち歩く代わりに、この金庫を経由して最新のデータをやり取りします。
*   **Pull (プル)：** 金庫から最新のデータを手元にダウンロードする
*   **Commit (コミット) / Push (プッシュ)：** 手元の最新状態を金庫にアップロードする

---

## 📍 2. 日々の作業の流れ（これを繰り返します！）

### 【作業を終わる時】データをインターネットの金庫に預ける
エディタ（Antigravity）などで作業を終えて、ファイルを保存（`Ctrl + S`）した後に行います。

1. **GitHub Desktop** を開く。
2. 左下の「Summary」の欄に、何をしたか簡単なメモを書く（例：「日報を書いた」「テストページ少し修正」）。
3. その下にある青いボタン **「Commit to main」** を押す。（変更を箱に詰める作業です）
4. 画面上部にある上向き矢印のボタン **「Push origin」** を押す。（インターネットの金庫へ運ぶ作業です）
※これでバックアップが完了し、別のPCで見れるようになります！

### 【別のPCで作業を始める時】最新データを引き継ぐ
会社に着いて作業を始める時や、家に帰ってきて続きをやる時に、一番最初に行います。

1. **GitHub Desktop** を開く。
2. 画面上部の **「Fetch origin」** を押して、金庫に新しいデータがないかをサーバーに確認する。
3. もし別のPCで作業した新しいデータがあれば、ボタンが **「Pull origin」** (下向き矢印) に変わります。それを押してダウンロードします。（これで手元のPCが最新状態になります）

## 📍 3. 新しいプロジェクト（フォルダ）を同期追加したい時
「website_test_0223」など、新しく作ったフォルダを同期したい場合に行う「一番最初の1回だけ」の設定です。

### 【ステップ1：自宅PC】インターネット上に新しい金庫を作る
1. エクスプローラーで、同期したい「英語名のフォルダ」を `...\ドキュメント\GitHub\` フォルダの直下に置く。
2. GitHub Desktop の左上から **「Add」 ＞ 「Add existing repository...」** を選ぶ。
3. 用意したフォルダを選択する。
4. 青い文字の **「create a repository」** ＞ 青いボタン **「Create repository」** を押す。
5. 画面上部の **「Publish repository」** を押す。（※非公開にするため "Keep this code private" のチェックは入れたままにします）

### 【ステップ2：会社PC】出来上がった金庫からデータを降ろしてくる
1. GitHub Desktop の上部メニューから **「File」 ＞ 「Clone repository...」** を選ぶ。
2. リストの中に「ステップ1で作った金庫の名前」があるので、それを選ぶ。
3. そのまま右下の青い **「Clone」** ボタンを押す。

---

## ⚠️ 4. 注意事項・ルール

*   **重いファイルは絶対に入れないこと！**
    動画、長時間の音声ファイル（.m4a等）、画像が大量に入ったWordやExcel、PDFなどのファイルは、GitHubに入れると重くなりシステムが壊れる原因になります。これらは会社指定の **Box** や **Googleドライブ** 等に保存してください。
*   **フォルダ名・ファイル名は必ず「英語」で！**
    日本語の名前をつけるとエラーや文字化けの原因になります。必ず「半角英数字とハイフン」（例：`copywriting-notes`）を使う約束にしましょう。
*   **アイコンの違いについて**
    *   🔒 **カギのマーク (Private):** 自分しか見れない安全な状態。お仕事の大事なデータはすべてこれ（初期設定）でOKです。
    *   📖 **本のマーク (Public):** 世界中の誰でも見れる状態。公開してもいい学習記録などに使います。
